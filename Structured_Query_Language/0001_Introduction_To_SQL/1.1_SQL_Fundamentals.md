# 1.1 SQL Fundamentals

SQL fundamentals is the foundation of everything in database engineering, backend systems, analytics engineering, data engineering, and modern software development.

Before learning complex queries, optimization, warehousing, or distributed databases, you must first understand what SQL actually is, why it exists, and where it is used in modern systems.

---

# 1.1.1 What is SQL

## Definition
SQL (Structured Query Language) is a standard language used to communicate with relational databases.

It allows users to:

- store data
- retrieve data
- modify data
- delete data
- define database structures
- control access permissions
- manage transactions

---

## Core Purpose of SQL

SQL is used to interact with relational database systems such as:

- MySQL
- PostgreSQL
- SQL Server
- Oracle
- SQLite
- MariaDB
- Snowflake
- BigQuery
- Redshift

---

## What SQL Can Do

### Data Retrieval
- SELECT queries
- searching records
- filtering data

### Data Insertion
- adding new records

### Data Modification
- updating existing records

### Data Deletion
- removing records

### Database Design
- creating tables
- defining relationships
- constraints

### Security
- permissions
- roles

### Transaction Management
- commit
- rollback

---

## SQL Categories

### DDL (Data Definition Language)
- CREATE
- ALTER
- DROP

### DML (Data Manipulation Language)
- INSERT
- UPDATE
- DELETE

### DQL (Data Query Language)
- SELECT

### DCL (Data Control Language)
- GRANT
- REVOKE

### TCL (Transaction Control Language)
- COMMIT
- RUNTIME
- SAVEPOINT

---

## Why SQL Exists

Without SQL:

- applications cannot persist data
- businesses cannot store records
- analytics cannot happen
- transactions cannot be processed
- reporting systems cannot function

SQL became the universal language for structured data systems.

# 1.1.4 SQL vs Programming Languages

SQL is not a general-purpose programming language.

It is a declarative language.

---

## SQL (Declarative)

You tell the database:

**what you want**

Example:

```sql
SELECT name
FROM employees
WHERE salary > 5000;
```

You do not tell it exactly how to retrieve data.

The database optimizer decides execution.

---

## Programming Languages (Procedural/OOP)

Examples:

- Python
- Java
- C++
- JavaScript
- Go

You tell the computer:

**how to perform tasks**

Example:

- loops
- variables
- algorithms
- classes
- APIs

---

## Major Differences

### SQL
- data-focused
- declarative
- database interaction

### Programming Languages
- application-focused
- procedural/object-oriented
- full software development

---

## SQL + Programming Languages Together

Modern applications often use:

- Python + PostgreSQL
- Java + MySQL
- Node.js + MongoDB/PostgreSQL
- C# + SQL Server

They work together.

# 1.1.5 SQL vs NoSQL

This is one of the most misunderstood comparisons.

It is not:

SQL good  
NoSQL bad

or

NoSQL good  
SQL bad

Different tools solve different problems.

---

## SQL Databases

Examples:

- PostgreSQL
- MySQL
- SQL Server
- Oracle

### Strengths
- ACID transactions
- structured relationships
- strong consistency
- complex queries

### Best For
- banking
- ERP
- SaaS
- transactional systems

---

## NoSQL Databases

Examples:

- MongoDB
- Cassandra
- Redis
- DynamoDB
- Neo4j

### Strengths
- horizontal scaling
- flexible schemas
- high throughput
- specialized workloads

### Best For
- caching
- large-scale distributed systems
- graph relationships
- document storage

---

## Hybrid Architecture

Modern systems often combine both:

- PostgreSQL → transactions
- Redis → caching
- MongoDB → documents
- Elasticsearch → search
